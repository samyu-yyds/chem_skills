# 化工企业报警分析报告生成

本技能是流程编排器，通过依次调用两个子技能完成核心任务，自身只负责参数规划、数据统计和提示词生成。

## 输入参数（由用户对话提供）

- `enterprise_name`：企业全称（必须从枚举值中选择）
- 诊断月份或日期区间（若未提供，默认生成上月报告）

## 执行流程

### 步骤 1：确定参数并创建工作目录

1. 调用 `获取系统日期时间` 技能，得到当前日期。
2. 计算 `start_date`、`end_date`（默认当月1日~昨日，或用户指定的完整月份）。
3. 确定工作目录：`/tmp/alarm_report_{企业简称}_{YYYYMM}`，例如 `/tmp/alarm_report_巍华_202604`。

### 步骤 2：获取全量报警数据

调用技能 **企业报警数据查询**，传入参数：

- `start_date` = 上述开始日期（格式 YYYY-MM-DD）
- `end_date` = 上述结束日期
- `enterprise_name` = 用户指定的企业名称
- `tmp_dir` = 步骤1的工作目录

该技能执行后，会在工作目录下生成 `all_alarms.json`，包含企业全量报警记录。

### 步骤 3：高风险报警判定

调用技能 **高风险报警判定（规则引擎版）**，传入参数：

- `alarm_record_file` = 工作目录下的 `all_alarms.json` 绝对路径
- `output_highrisk_file` = 工作目录下的 `high_risk_alarms.json` 绝对路径

该技能执行后，`high_risk_alarms.json` 已生成，所有字段已转为中文，且每条记录均包含 `高风险原因` 列表。

### 步骤 4：统计基础数据与准备报告素材

读取 `all_alarms.json`，统计以下数据：

- `total_alarms`：总报警条数
- `data_alarms`：数据报警数（`alarm_type == "1"` 或非 `"2"` 的报警）
- `video_alarms`：视频报警数（`alarm_type == "2"`）
- `closure_rate`：闭环率 = 已处置报警数（`close_time` 非空）÷ 总数 × 100%

读取 `high_risk_alarms.json`，获取：

- `high_risk_count`：高风险报警总数
- `high_risk_details`：高风险报警完整列表（已含中文字段及 `高风险原因`）

你说得对，智能体看技能描述不需要“你调用大模型”这种元指令，直接说明任务内容和输出要求就行。下面把步骤 5 改成更自然的智能体指令。

### 步骤 5：分析高风险报警并生成报告

你现在已经拥有了完整的数据：
- 全量报警统计（总数、数据报警数、视频报警数、闭环率）
- 高风险报警列表（`high_risk_alarms.json`），每条包含中文字段和 `高风险原因`
- 报警分级参考（仅供报告生成时参考引用）
    本次诊断聚焦高风险报警（P0），定义如下：
    - P0 高风险：可能导致事故的重大隐患，需立即电话通知企业（15分钟内）
    - P1 中风险：需纳入月度检查清单，周期性分析
    - P2 低风险：批量归档，定期清理
    报告应重点分析 P0 项，P1/P2 仅在闭环率统计等章节简要提及。
你需要两件事：**深入分析高风险报警**，并**按模板生成正式报告**。

#### 5.1 高风险报警分析要求

对每一条高风险报警，做以下两件事：

**1）完善和润色“高风险判断依据”**
- 代码已经给出了初步的判定原因（如“规则一：A级报警未恢复”）。
- 你需要结合该报警的 `报警详情`、`报警等级`、`检测类型`、`区域名称`、`报警值`、`阈值` 等字段，把原因展开成一句或一段专业的、可放在报告里的话。
- 例如：代码给出“规则一：A级报警未恢复”，结合数据可改写为“该报警为A级严重报警，监测点位为氟化氢储槽温度，自2026-04-01 10:04触发以来一直未恢复，构成重大风险敞口”。

**2）分析“风险危害”**
- 基于该报警涉及的介质（从 `报警对象名称`、`检测类型` 推断）、所在区域（是否为重大危险源区域）、报警严重程度，分析如果失控可能导致的后果。
- 需要使用化工安全专业知识，写出具体的、针对性的危害描述，不能泛泛而谈。
- 例如：氟化氢温度高报警 → 可能导致储槽压力升高、泄漏风险，氟化氢剧毒，泄漏可造成严重中毒和环境污染。

#### 5.2 报告生成要求

按以下模板生成完整的报告 Word 文档。

**报告封面：**
```
xxx经济技术开发区
{企业名称}企业报警分析报告
({开始日期} 至 {结束日期})

北京xxxx科技有限公司
{当前日期}
```

**目录：**
- 一、报警诊断范围及方式
- 二、风险分析结果及闭环情况
- 三、巡查指引单
- 四、附件
- 五、事故分享

**正文一、报警诊断范围及方式**
（一）诊断范围
本次诊断对象为{企业名称}，数据来源为园区综合监管平台{月份}月报警全量数据，涵盖多类核心数据集：数据报警{数据报警数}条、视频报警{视频报警数}条、中化协驻企报告（{企业名称}驻点服务报告）、特殊作业等，具体详见附件1。

（二）诊断方式
本次诊断采用“大模型全量数据分析＋工程师人工复核”两阶段方法。

**二、风险分析结果及闭环情况**
1、本次诊断共发现{高风险报警总数}项高风险报警，具体高风险报警情况如下：

| 序号 | 具体报警描述 | 报警类型 | 报警原因 | 高风险判断依据 | 风险危害 |
|------|-------------|---------|---------|---------------|---------|
| 1 | 根据报警详情概括一条简要描述 | 点位类型或报警类型 | 你分析出的根本原因 | 你在5.1中完善后的判断依据 | 你在5.1中分析的风险危害 |
| 2 | ... | ... | ... | ... | ... |

2、本月闭环率{闭环率}%，整改率100%，具体情况如下：
1）企业反馈单照片；         2）片管现场指导整改照片；

**三、巡查指引单**
根据本次高风险报警暴露的问题，列出需要重点巡查的点位、区域、设备类型及巡查要点，逐条说明。

**四、附件**
附件1：全量报警数据文件路径 `{工作目录}/all_alarms.json`
附件2：高风险报警详情文件路径 `{工作目录}/high_risk_alarms.json`

**五、事故分享**
同类隐患在其它地方出现过事故：
结合高风险报警中涉及的危险介质、超标情况，简述2~3个相关化工行业典型事故案例，说明失控后果和教训。

#### 5.3 输出要求
- 最终生成 Word 文档（`.docx`），保存到 `{工作目录}/{企业名称}_报警分析报告_{YYYYMM}.docx`。
- 报告必须使用真实数据，绝对不能编造数字或案例。
- 语言正式、专业、客观，符合安全生产监管部门报告风格。

### 步骤 6：交付结果
- 告知用户报告生成成功，提供文件路径。
- 询问是否需要清理临时数据。

