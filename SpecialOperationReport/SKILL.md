# 化工园区特殊作业运营调研技能

此技能提供了一种结构化的方法来开展化工园区特殊作业方面的运营调研，它使用“任务”工具来生成调研子代理。它强调规划、高效委派任务以及系统地综合调研结果。

## 何时使用此技能

以下情况可以使用此技能：
- 从多个维度分析化工园区特殊作业运营情况
- 从化工园区特殊作业系统查询多方面信息
- 对化工园区特殊作业系统进行信息汇总并进行数据分析
- 撰写资料翔实、引用清晰的化工特殊作业运营报告

## 调研流程

### 步骤 1：创建并保存调研计划

1. **获取当前的日期信息** - 依赖`获取系统的日期时间技能`

2. **创建调研文件夹** - 将所有调研文件整理到一个与当前工作目录关联的专用文件夹中：
```
mkdir [YOUR_LOCALWORK_DIR]/research_[topic_name]
```
这样可以保持文件井然有序，避免工作目录杂乱无章。

注意：将`[YOUR_LOCALWORK_DIR]`替换为当前工作目录的绝对路径

3. **分析调研问题** - 将其分解为彼此独立、互不重叠的子任务

4. **编写调研计划文件** - 使用 `write_file` 工具在当前工作目录下创建 `[YOUR_LOCALWORK_DIR]/research_[topic_name]/research_plan.md` 文件，其中包含：
- 主要调研的问题
- 1-5个待调研的具体子任务
- 每个子任务的预期要查询的信息
- 怎样将各子任务调研结果进行汇总

**子任务规划指南：**:

   - **数智诊断子任务**
      - 查询园区整体特殊作业的报备作业与实际作业数量情况，按照用户问题中限制的时间范围分析几个关键指标【计划特殊作业总量, 实际开展活动总量, 整体作业完备率，实际作业总量的环比提升比例】
      - 统计化工企业在作业活动的表现情况，按照用户问题中限制的时间范围分析几个关键指标【行特殊作业数量最多的前五名企业分布，化工企业作业完备率为100%的企业数量以及企业列表，作业完备率小于100%的企业数量以及出现偏差的分析，统计企业作业完备率最低的前三家企业列表】
      - 查询特殊作业类型分布，按照用户问题中限制的时间范围分析几个关键指标【作业数量排名前三的作业类型分布, 统计排名前三的作业类型中企业维保、承包商各自的作业数量和占比情况】

   - **园区线上监管子任务**
      - 查询企业报警数据, 按照用户问题中限制的时间范围分析几个关键指标【报警最多的前三家企业】

   - **化工企业开停车子任务**
      - 查询化工企业设备开停车数据, 按照用户问题中限制的时间范围分析几个关键指标【是否存在设备开、停车行为】

- **重要提示**：
   - **关键行为要求**：
   （1）`[YOUR_LOCALWORK_DIR]`为你当前工作目录的绝对路径；
   （2）以上子任务的执行依赖于 `化工园区特殊作业数据查询技能` ；
   （3）不允许使用合理的假设创建示例数据来完成报告，必须获取园区特殊作业真实的数据；
   （4）对子任务的的各项指标进行查询时，需要分步查询，不要一步同时查询多个指标;

   - **业务指标解释**：
      （1）在特殊作业系统中用字段`作业状态`表示作业的实际流转状态，取值范围：【计划未正常关闭，计划未执行，关闭，作业中，撤销，填报】；
      （2）实际开展的作业：用字段`作业状态`取值：【计划未正常关闭、作业中、关闭】表示，这三个取值都可以用来表示实际开展的活动；
      （3）计划作业：用字段`作业状态`取值：【计划未正常关闭，计划未执行，关闭，作业中，撤销，填报】表示，这六个取值都可以表示计划的作业活动；
      （4）作业完备率：用实际作业量与计划作业量的比值表示，企业作业完备率表示企业实际的作业活动/企业计划的作业活动；

### 步骤 2：子任务的委派

1. **使用 `task` 工具** 在当前工作目录下创建一个子任务，并包含：
- 清晰明确的调研问题（不使用缩写）
- 任务说明将子任务的调研结果写入文件：`[YOUR_LOCALWORK_DIR]/research_[topic_name]/reasearch_[子任务名称]/findings_[子任务].md`

2. **最多可同时运行2个子任务**，以提高研究效率

3. **若使用数据查询工具，则数据查询完成后，使用 `write_file` 工具将查询的结果做结构化Markdown格式文件保存，方便阅读和理解，路径为`[YOUR_LOCALWORK_DIR]/research_[topic_name]/research_[子任务名称]/findings_[子任务].md` 文件**

注意：`[YOUR_LOCALWORK_DIR]`为你当前工作目录的绝对路径

### 步骤 3：综合研究结果

所有子任务完成后：
1. **分析本地保存的子任务调研结果文件**：

- 首先运行 `ls [YOUR_LOCALWORK_DIR]/research_[topic_name]` 查看创建的文件
- 然后使用 `read_file` 并指定**文件路径**（例如，`research_[topic_name]/findings_*.md`）
- 利用 `Markdown结构化数据分析技能` 对各个子任务生成的Markdown文件进行完整的分析和解读

- **重要提示**：`read_file` 仅用于读取本地文件，不要读取 URL

2. **综合信息** - 创建一份全面的报告，该报告应：

- 直接回答原始问题
- 整合所有子任务的见解
- 指出任何不足或局限性

3. **撰写最终报告**
- 按照用户的要求，使用 `write_file` 创建指定格式的报告文件 例如：`[YOUR_LOCALWORK_DIR]/research_[topic_name]/[你的报告名称].docx`

注意：`[YOUR_LOCALWORK_DIR]`为你当前工作目录的绝对路径

## 最佳实践

- **委派任务前做好计划** - 始终先编写 research_plan.md 文件
- **子任务查询** - 统计子任务的各项指标时，需要分布查询，不要一步同时查询多个指标
- **明确子主题** - 确保每个子代理都拥有明确且互不重叠的范围
- **基于文件的沟通** - 要求子代理人将调查结果保存到文件中，而不是直接提交
- **系统性综合** - 在撰写最终回复前，阅读所有调查结果文件
- **适时停止** - 不要过度研究；通常每个子主题进行 3-5 次搜索就足够了。
